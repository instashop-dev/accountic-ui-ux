The revised ICAI Code of Ethics took effect on **1 April 2026**. Buried in the management consultancy services section is a paragraph that every practicing CA who uses an AI tool — for notice drafting, for ledger classification, for document summarisation — needs to know by name.

## The operative language

The Code formally recognises AI tooling deployed by a Chartered Accountant in the conduct of professional services as **management consultancy services (MCS)**. This is not a revolution; it's a classification. But it comes with consequences.

The two most important:

1. **The CA retains professional responsibility for the output.** You can't disclaim the draft because "the AI wrote it." An AI-assisted reply that misstates law, misattributes a case, or misreads a client's position is a reply you signed.
2. **"Application of professional judgment" is the line.** The Code doesn't prohibit AI. It insists that the judgment applied to the final output be the CA's — demonstrable on the record, reviewable in peer review, and defensible in an ICAI disciplinary proceeding.

## What this changes in practice

For most firms, the practical change is modest but specific:

- **Workflow records.** You should be able to show, for any given matter, what the AI produced, what you changed, and why. An AI tool that doesn't give you an audit log is a problem.
- **Citation verification.** The Code's "professional judgment" standard is hard to defend if the draft you signed contains a hallucinated section or a fabricated case. Verification has moved from nice-to-have to table stakes.
- **Confidentiality.** Client data fed into a consumer AI tool, sent abroad, with no DPA and training-on-by-default, is a straight MCS compliance failure. It's also, separately, a DPDP failure.
- **Peer review exposure.** Peer reviewers will, from 2026 onward, start asking about AI tooling used. Having a clean answer — name of tool, residency, DPA, audit log — is better than improvising.

## What this *doesn't* change

The Code does not prohibit AI. It does not require ICAI pre-approval of AI tools. It does not restrict AI use to a particular firm size or service line. It sets a professional judgment standard that any good CA was already applying to any drafted output, human or otherwise.

If you already read every draft before it leaves the firm, you are already substantially compliant. The Code just makes the workflow requirements around *how* you do that reading a little more formal.
